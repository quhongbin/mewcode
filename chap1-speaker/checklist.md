# MewCode Checklist

> 每一项通过运行代码或观察行为来验证，聚焦系统行为。

## 实现完整性

### 启动与界面
- [ ] 启动程序后，终端显示欢迎信息和输入提示符（验证：运行 `go run ./cmd/mewcode`，观察界面显示欢迎文本和可输入区域）
- [ ] 输入框可接收文本输入（验证：键入字符，观察输入框显示对应内容）

### 对话功能
- [ ] 用户输入后按 Enter，AI 开始生成回复（验证：输入问题后按 Enter，观察 AI 开始响应）
- [ ] AI 回复以流式方式逐字显示（验证：观察长回复的生成过程，可见字符逐步出现，非一次性输出）
- [ ] 多轮对话中 AI 能引用之前内容（验证：第一轮说"我叫小明"，第二轮问"我叫什么"，AI 回答"小明"）

### 配置管理
- [ ] 通过 YAML 配置文件指定供应商信息（验证：修改配置文件中的 model 字段，重启后观察请求使用新模型）
- [ ] 配置文件包含多个供应商时，可通过 name 切换（验证：配置两个供应商，修改 active_provider 字段后重启，观察请求发往不同供应商）
- [ ] 配置缺失必填字段时启动报错并退出（验证：删除 api_key 字段后启动，观察错误提示并退出）

### API 协议支持
- [ ] Anthropic Claude 协议正常工作（验证：配置 protocol: anthropic，发送消息，收到回复）
- [ ] Anthropic extended thinking 功能正常（验证：配置 thinking: true，观察响应中包含思考过程内容）
- [ ] OpenAI 协议正常工作（验证：配置 protocol: openai，发送消息，收到回复）

### 交互功能
- [ ] 输入 `/exit` 或按 Ctrl+C 后程序优雅退出（验证：执行退出操作，程序正常结束，无残留进程）
- [ ] Alt+Enter 可插入换行，Enter 提交（验证：输入多行文本，观察换行正确插入，按 Enter 后提交）
- [ ] 请求失败时显示错误信息，程序不退出（验证：配置错误的 API Key，发送消息，观察错误提示，继续输入新消息）
- [ ] 请求发出后显示计时指示，结束后显示总耗时（验证：发送消息，观察 "Imagining… (Xs)" 显示，回复完成后观察 "Done in Xs"）

### 性能与稳定性
- [ ] 流式输出过程中 TUI 保持流畅（验证：长回复生成过程中观察界面，无卡顿或闪烁）

## 集成

- [ ] Config 模块正确解析配置并传递给 Provider（验证：修改配置后观察 Provider 使用新配置）
- [ ] Provider 模块正确实现 Stream 接口，Conversation 可正常调用（验证：发送消息后观察流式响应正常返回）
- [ ] Conversation 模块正确维护对话历史，多轮对话上下文正确（验证：多轮对话中 AI 能引用之前内容）
- [ ] TUI 模块正确调用 Conversation，流式响应实时显示（验证：观察流式输出过程，字符逐步出现）
- [ ] 错误从 Provider 层正确传递到 TUI 层显示（验证：触发错误后观察 TUI 显示错误信息）

## 编译与测试

- [ ] 项目编译无错误（验证：运行 `go build ./...`，无报错输出）
- [ ] 所有单元测试通过（验证：运行 `go test ./...`，全部 PASS）
- [ ] Config 模块测试覆盖正常和异常场景（验证：运行 `go test ./internal/config/...`，全部通过）
- [ ] Conversation 模块测试覆盖多轮对话场景（验证：运行 `go test ./internal/conversation/...`，全部通过）
- [ ] Provider 模块测试覆盖 SSE 解析逻辑（验证：运行 `go test ./internal/provider/...`，全部通过）

## 端到端场景

### 场景 1：完整对话流程
- [ ] 启动程序 → 输入问题 → 观察流式响应 → 继续多轮对话 → 输入 `/exit` 退出（验证：整个流程无报错，多轮对话上下文正确，退出干净）

### 场景 2：错误恢复
- [ ] 配置错误 API Key → 启动程序 → 输入问题 → 观察错误提示 → 继续输入新问题（验证：错误后程序不退出，可继续对话）

### 场景 3：多行输入
- [ ] 启动程序 → 输入多行文本（使用 Alt+Enter）→ 按 Enter 提交 → 观察 AI 正确理解多行内容（验证：多行输入正确提交，AI 响应正确）

### 场景 4：供应商切换
- [ ] 配置两个供应商 → 使用第一个供应商对话 → 修改配置切换到第二个供应商 → 重启程序 → 继续对话（验证：切换后请求发往新供应商，对话正常）

### 场景 5：Extended Thinking
- [ ] 配置 Anthropic 协议并启用 thinking → 输入复杂问题 → 观察思考过程显示 → 观察最终回复（验证：思考过程和回复分别显示，内容相关）
