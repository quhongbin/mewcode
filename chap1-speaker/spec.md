# MewCode Spec

## 背景

当前项目是空的，需要从零构建 MewCode —— 一个命令行 AI 编程助手。目标是提供类似 Claude Code 的终端交互体验，让用户能在命令行中与 AI 进行多轮对话。

## 目标

- **G1**: 提供终端交互式对话界面，用户可输入问题并获得 AI 回复
- **G2**: 支持多轮对话，AI 能记住完整对话历史
- **G3**: 流式输出回复内容，逐字显示而非等待完整响应
- **G4**: 支持 Anthropic Claude 和 OpenAI 两种 API 后端，通过配置切换
- **G5**: 支持 Claude 的 extended thinking 功能
- **G6**: Provider 层抽象为统一接口，便于后续扩展新的 API 后端

## 功能需求

- **F1**: 启动 MewCode 后进入交互式对话界面，显示欢迎信息和输入提示符
- **F2**: 用户输入文本后按 Enter 提交，AI 开始生成回复
- **F3**: AI 回复以流式方式逐字显示在终端，用户可实时看到生成过程
- **F4**: 支持多轮对话，每次请求携带完整对话历史
- **F5**: 通过 YAML 配置文件指定 LLM 供应商，包含六个字段：name（供应商标识）、protocol（协议类型：anthropic 或 openai）、model（模型名称）、base_url（请求地址）、api_key（认证密钥）、thinking（是否启用扩展思考，可选）
- **F6**: 支持 Anthropic Claude API，包括 extended thinking 模式
- **F7**: 支持 OpenAI API
- **F8**: 配置文件支持定义多个供应商配置，通过 name 字段选择当前使用的供应商
- **F9**: 提供退出命令（如 `/exit` 或 Ctrl+C），优雅结束程序
- **F10**: 输入与提交 — 用户在输入框键入文本，可用 Alt+Enter 插入换行进行多行编辑；按 Enter 提交。提交后清空输入框，界面进入等待/流式状态，期间不接受新的提交，直至本轮回复结束
- **F11**: 错误反馈 — 当请求失败（鉴权失败、限流、网络中断、模型不存在等）时，在对话区以可区分的样式展示错误信息，程序不退出，用户可继续下一轮对话
- **F12**: 响应计时 — 自请求发出即启动计时，以"进行中"指示实时显示已用秒数（形如 "Imagining… (5s)"，秒数随时间递增）；收到首个增量后继续计时；本轮回复结束后定型显示该轮总耗时

## 非功能需求

- **N1**: 流式响应延迟 — 从发送请求到首个字符显示，延迟不超过网络往返时间加 500ms
- **N2**: TUI 流畅性 — 流式输出过程中界面保持响应，不卡顿、不闪烁
- **N3**: 配置验证 — 启动时校验配置文件格式和必填字段，缺失或格式错误时给出明确提示并退出
- **N4**: 错误恢复 — 单次请求失败不影响后续对话，程序保持稳定运行
- **N5**: 可扩展性 — Provider 接口设计使新增 API 后端只需实现接口，无需修改核心逻辑
- **N6**: 终端兼容性 — 支持主流终端模拟器（Windows Terminal、iTerm2、GNOME Terminal 等）
- **N7**: API Key 安全 — 配置文件中的 API Key 不以明文形式出现在日志或错误输出中
- **N8**: 代码可测试性 — 核心逻辑（Provider、配置解析、对话管理）与 UI 层解耦，支持单元测试

## 不做的事

- **E1**: Tool Use / Function Calling — 不实现 AI 调用外部工具的能力
- **E2**: 文件读写操作 — 不实现读取、创建、编辑本地文件的功能
- **E3**: 代码编辑能力 — 不实现代码搜索、修改、diff 展示等 agent 功能
- **E4**: 上下文管理策略 — 不实现对话历史的截断、压缩或 token 计数
- **E5**: 多模型同时对比 — 不支持单次请求同时调用多个供应商
- **E6**: 图形界面 — 不实现 Web UI 或桌面应用
- **E7**: 对话持久化 — 不保存对话历史到本地文件，重启后历史清空
- **E8**: 自定义主题/样式 — 不提供 TUI 配色或布局的自定义选项
- **E9**: 插件系统 — 不实现第三方扩展机制
- **E10**: 语音输入/输出 — 不涉及音频处理

## 验收标准

- **AC1**: 启动 MewCode 后，终端显示欢迎信息和输入提示符，用户可输入文本（验证：运行程序，观察界面）
- **AC2**: 用户输入文本后按 Enter，AI 开始生成回复（验证：输入问题，观察 AI 响应）
- **AC3**: AI 回复以流式方式逐字显示，非等待完整响应后一次性输出（验证：观察长回复的生成过程，可见字符逐步出现）
- **AC4**: 连续多轮对话中，AI 能引用之前对话内容（验证：第一轮说"我叫小明"，第二轮问"我叫什么"，AI 回答正确）
- **AC5**: 通过 YAML 配置文件指定供应商信息，程序正确读取并使用（验证：修改配置文件后重启，观察请求使用新配置）
- **AC6**: 使用 Anthropic Claude 协议时，配置 thinking: true 可启用 extended thinking，回复中包含思考过程（验证：配置 thinking: true，观察响应中包含 thinking 内容）
- **AC7**: 使用 OpenAI 协议时，程序正常调用 API 并返回回复（验证：配置 protocol: openai，发送消息，收到回复）
- **AC8**: 配置文件包含多个供应商配置时，可通过 name 字段选择使用哪个（验证：配置两个供应商，切换 name 后观察请求发往不同供应商）
- **AC9**: 输入 `/exit` 或按 Ctrl+C 后程序优雅退出（验证：执行退出操作，程序正常结束，无残留进程）
- **AC10**: 输入框中按 Alt+Enter 可插入换行，按 Enter 提交（验证：输入多行文本，观察换行正确插入，Enter 后提交）
- **AC11**: 请求失败时（如 API Key 错误、网络中断），对话区显示错误信息，程序不退出（验证：配置错误的 API Key，发送消息，观察错误提示，继续输入新消息）
- **AC12**: 请求发出后显示计时指示（如 "Imagining… (5s)"），回复结束后显示总耗时（验证：发送消息，观察计时显示，回复完成后观察最终耗时）
- **AC13**: 配置文件缺失必填字段时，程序启动报错并退出（验证：删除 api_key 字段后启动，观察错误提示）
- **AC14**: 流式输出过程中 TUI 保持响应，不卡顿（验证：长回复生成过程中尝试滚动或观察界面流畅度）
